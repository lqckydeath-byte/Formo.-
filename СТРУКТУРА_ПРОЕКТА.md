### Объяснение

| Папка | Назначение |
|-------|-----------|
| `features/` | Каждый feature (auth, home, surveys) - независимый модуль |
| `pages/` | React компоненты страниц |
| `shared/` | Код, который используется в нескольких features |
| `api/` | Единое место для API запросов |

### Ключевые Файлы

| Файл | Роль | Размер |
|------|------|--------|
| `App.js` | Конфигурация маршрутов и Protected Routes | ~40 LOC |
| `Home.jsx` | Главная страница (поиск, шаблоны, логотип) | ~450 LOC |
| `SurveyEditor.jsx` | Редактор опросов (вопросы, настройки) | ~750 LOC |
| `client.js` | API клиент для общения с backend | ~50 LOC |

## 🖥️ SERVER - Backend Structure

```
server/
├── config/
│   └── db.js              ← MongoDB подключение
│
├── models/
│   ├── User.js            ← Схема пользователя (bcrypt, JWT)
│   ├── Survey.js          ← Схема опроса (questions, metadata)
│   └── Answer.js          ← Схема ответов (responses)
│
├── routes/
│   ├── auth.js            ← POST /auth/register, /auth/login
│   ├── surveys.js         ← CRUD операции с опросами
│   └── answers.js         ← Сохранение и получение ответов
│
├── index.js               ← 🎯 ГЛАВНЫЙ ФАЙЛ СЕРВЕРА
├── .env.example           ← Пример переменных окружения
├── package.json
├── package-lock.json
└── node_modules/          ← (gitignore)
```

### Файл .env.example

```env
# Server Port
PORT=5000

# Database
MONGODB_URI=mongodb://localhost:27017/formo

# JWT Secret
JWT_SECRET=your-secret-key-here

# Environment
NODE_ENV=development
```

### Ключевые Эндпоинты

```
POST   /api/auth/register     ← Регистрация
POST   /api/auth/login        ← Вход
GET    /api/surveys           ← Все опросы пользователя
POST   /api/surveys           ← Создать опрос
GET    /api/surveys/:id       ← Получить конкретный
PUT    /api/surveys/:id       ← Обновить опрос
DELETE /api/surveys/:id       ← Удалить опрос
POST   /api/answers           ← Сохранить ответы
GET    /api/answers/survey/:id ← Получить все ответы
```

## 🔐 Безопасность

### На фронтенде

```javascript
// Protected Routes в App.js
<Route path="/" element={<ProtectedRoute><Home /></ProtectedRoute>} />

// JWT токен в localStorage
localStorage.setItem("token", data.token);
```

### На бэкенде

```javascript
// Хеширование паролей в User.js
userSchema.pre('save', async function(next) {
  const salt = await bcrypt.genSalt(10);
  this.password = await bcrypt.hash(this.password, salt);
});

// Генерация JWT в auth.js
const token = jwt.sign({ id: user._id }, JWT_SECRET, { expiresIn: '7d' });
```

## 🎨 CSS-in-JS Стили

Все стили встроены прямо в компоненты (внутри `<style>`):

```javascript
<style>{`
  .navbar { display: flex; padding: 12px; }
  .container { max-width: 1200px; margin: 0 auto; }
  
  @media (max-width: 768px) {
    .navbar { padding: 8px; }
  }
  
  @keyframes fadeIn {
    from { opacity: 0; }
    to { opacity: 1; }
  }
`}</style>
```

**Преимущества:**
- ✅ Нет отдельных CSS файлов
- ✅ Стили скопированы с компонентом
- ✅ Легко модифицировать
- ✅ Автоматическая оптимизация в production

## ✅ По Критериям Оценки

### 1. Структура кода ✅
- Feature-based архитектура
- Каждый компонент в отдельном файле
- Shared код в shared/ папке

### 2. Навигация и маршруты ✅
- React Router в App.js
- Protected routes для защиты
- Все маршруты централизованы

### 3. Компоненты ✅
- Функциональные компоненты с Hooks
- Props для передачи данных
- State для управления состоянием

### 4. API интеграция ✅
- Centralised API client (client.js)
- Все запросы в одном месте
- Error handling везде

### 5. Безопасность ✅
- bcrypt для паролей
- JWT для аутентификации
- Protected routes на фронте

### 6. Стили ✅
- CSS-in-JS (встроенные стили)
- Responsive design (media queries)
- Анимации и transitions
